# Demo Recording Instructions

## Pre-Recording Checklist

- [ ] Backend is running and healthy (http://localhost:3001/health)
- [ ] Frontend is running (http://localhost:5173)
- [ ] Gemini API key is configured (or mock mode is working)
- [ ] Sample user data is loaded (backend/data/users.json)
- [ ] All browser tabs are closed except demo-related ones
- [ ] Microphone is tested and working
- [ ] Screen recording software is set up (Loom, OBS, or similar)
- [ ] Demo script is reviewed and ready

## Recording Setup

### Screen Recording Software Options

1. **Loom** (Recommended)
   - Free tier available
   - Easy sharing and embedding
   - Built-in editing tools

2. **OBS Studio** (Free, Open Source)
   - More control over recording settings
   - Requires manual upload

3. **Zoom** (If you have access)
   - Record meeting with yourself
   - Good audio quality

4. **Windows Game Bar** (Windows 10/11)
   - Press Win+G to open
   - Built-in screen recording

### Audio Setup

- Use a good microphone (headset or USB mic)
- Test audio levels before recording
- Minimize background noise
- Speak clearly and at a moderate pace
- Use a quiet room

### Screen Setup

- **Resolution**: 1920x1080 recommended
- **Browser**: Use Chrome or Edge (for voice widget)
- **Zoom**: 100% or 125% (readable but not too large)
- **Theme**: Light mode for better visibility

## Recording Tips

### Before Starting

1. **Close Unnecessary Apps**: Close email, Slack, etc.
2. **Clear Browser Cache**: Ensure fresh state
3. **Test Run**: Do a 30-second test recording
4. **Check Audio**: Verify microphone is working
5. **Prepare Notes**: Have demo script open in another window

### During Recording

1. **Speak Clearly**: Enunciate, moderate pace
2. **Use Cursor**: Point to important elements
3. **Pause When Needed**: It's okay to pause and think
4. **Show Errors Gracefully**: If something fails, explain what should happen
5. **Stay Calm**: Take deep breaths between sections

## Post-Recording

### Editing (Optional)

1. **Trim**: Remove long pauses or mistakes
2. **Add Titles**: Add section titles if helpful
3. **Speed Up**: Slightly speed up slow sections (1.25x)
4. **Add Captions**: Consider adding subtitles

### Upload

1. **Loom**: Upload directly, get shareable link
2. **YouTube**: Upload as unlisted, share link
3. **Vimeo**: Upload, get embed code

### Sharing

- Include link in submission email/document
- Test the link before sharing
- Provide a brief description of what's in the video

## Backup Plan

If recording fails or has issues:

1. **Screenshots**: Take screenshots of key features
2. **GIFs**: Create animated GIFs of interactions
3. **Written Walkthrough**: Provide detailed written explanation
4. **Live Demo**: Offer to do a live demo if possible

## Time Management

- **Target**: 8-10 minutes
- **Maximum**: 12 minutes (if needed)
- **Minimum**: 7 minutes (if concise)

Breakdown:
- Intro: 30s
- Problem: 60s
- Architecture: 2:00
- Code: 2:00
- Live Demo: 2:00
- Testing/QA: 1:00
- Q&A/Next Steps: 1:00

**Remember**: The goal is to demonstrate your understanding of the system, not perfection. It's okay to make minor mistakes - just explain what should happen.
